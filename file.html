Final Review for Our Project

Overall Summary of Literature Review

Our research focuses on Student Performance Intelligence platforms that combine multi-source data ingestion, data preprocessing, predictive analytics, and actionable dashboards. These systems support schools, teachers, and parents with insights and timely alerts.

  • Data Integration: Importing PAT, SAT, and attendance data securely from schools.
  • Preprocessing: Cleaning, normalizing, and handling missing data for model readiness.
  • Predictive Analytics: ML and statistical models to predict student performance and risk.
  • Dashboards: Role-based dashboards with risk indicators for stakeholders.
  • Engagement: Automated alerts and messaging for parents and teachers.

We plan to build the mobile experience using Android via Apache Cordova, leverage Firebase Cloud Messaging for push notifications, and implement core logic in JavaScript with a modern web stack.

Solution Approach

  • a) Import: PAT, SAT, and attendance data from schools.
  • b) Preprocess: Clean, normalize, and handle missing data.
  • c) Predict: Use ML and statistical models to estimate outcomes and risk.
  • d) Dashboards: Risk indicators for admins, teachers, students, parents.
  • e) Alerts: Automated app notifications for parents and teachers.
  • Tech: Android (Apache Cordova), Firebase Cloud Messaging, JavaScript core logic.

Future Scope

  • Adaptive Learning: Personalized recommendations and study plans.
  • Offline Sync: Data capture with background synchronization.
  • Advanced Explainability: SHAP/LIME for model transparency.
  • Data Governance: Role-based access, audit logs, and compliance.

Tasks Not Possible

  • Guaranteeing 100% prediction accuracy across diverse cohorts.
  • Full offline alerts without network connectivity (push requires internet).
  • Real-time analytics on extremely low-end devices without cloud support.

We Can Work With

  • Web/mobile: HTML, CSS, JavaScript, React/Vue, Cordova (Android)
  • Backend: Node.js/Express, Python (for ML services)
  • Databases: PostgreSQL/MySQL, Firebase/Firestore for messaging state
  • Security: JWT, TLS, encryption at rest/in transit
  • Cloud: Firebase, AWS, Azure; FCM for push notifications
